# #58 - [MVP 1] Petabase — Veterinary PIMS MVP

## Problem
Veterinary clinics in Thailand lack an affordable, modern PIMS (Practice Information Management System) that supports Thai language, mobile-first workflows, and local business needs. Existing solutions are either too expensive, lack localization, or have outdated UX. This MVP establishes the foundational data platform for every veterinary clinic.

## Requirements

### Phase 1A: UI Prototype (Mock Data Only)
1. **Dashboard + Auth** — Clinic overview, today's appointments, quick stats, recent activity, login/register/forgot-password UI (mock auth)
2. **Patient Management** — Searchable/filterable patient list, patient profile with photo/owner card/medical history timeline/documents, owner CRUD with linked pets
3. **Appointment Scheduling** — Calendar view (day/week/month), time slot grid, color-coded by type/status, appointment form, status flow (scheduled→confirmed→in-progress→completed/cancelled/no-show), filterable list view
4. **Consultation / SOAP Notes** — Consultation flow from appointment or patient profile, SOAP note form (Subjective/Objective/Assessment/Plan), prescription (medications/dosage/frequency/duration), consultation history per patient, print-friendly layout
5. **Billing & Invoicing** — Invoice creation from consultation, line items (services/products/charges), discount/tax support, invoice list with status filters (draft/sent/paid/overdue), payment recording (cash/card/transfer), daily/weekly/monthly summary
6. **Organization & User Management** — Org settings (name/logo/contact/branches), branch CRUD, staff list with role badges and invite, role overview (Admin/Doctor/Nurse/Receptionist), permission matrix view

### Phase 1B: Backend Implementation
7. **Auth + Org + User + Role + Permission API** — Email/password auth, JWT tokens, password reset, org/branch CRUD, user CRUD with invite/activate/deactivate, RBAC with predefined roles and endpoint guards *(consolidates #23-#27)*
8. **Patient Management API** — Patient CRUD, search, photo upload, owner CRUD, owner-pet linking, medical history query
9. **Appointment Scheduling API** — Appointment CRUD, status management, doctor availability/schedule, slot generation, double-booking prevention, calendar query, reminder placeholders
10. **Consultation / SOAP Notes API** — Consultation CRUD linked to appointment/patient, SOAP note fields, prescription creation, medical record auto-generation, document upload
11. **Billing & Invoicing API** — Invoice CRUD from consultation or manual, payment recording with partial payments, revenue aggregation, outstanding balance queries

### Phase 2: Post-MVP
12. **Inventory Management** — Product catalog, stock per branch, adjustments/transfers, usage logging, low-stock alerts, expiry tracking
13. **Reporting & Analytics** — Revenue/patient visit/appointment/doctor performance reports, dashboard analytics with charts, PDF/CSV export

## Success Criteria
- [ ] Phase 1A: All 6 modules render with mock data and navigation
- [ ] Phase 1A: Responsive design working on mobile and desktop
- [ ] Phase 1B: All API endpoints functional with RBAC
- [ ] Phase 1B: UI wired to real API (mock data removed)
